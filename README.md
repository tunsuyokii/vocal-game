# Vocal Ball — вокальная распевка

2D-платформер: шарик движется сам, препятствия проходимы только при пении нужной ноты (До–Ре–Ми–Фа–Соль–Ля–Си). Управление голосом через микрофон.

## Важно: порт 3000

**Порт 3000 занят другим приложением.** Локальный сервер для этого проекта **нельзя** запускать на порту 3000 — используйте любой другой порт (например 3333 или 8080).

## Запуск локально

1. Поднимите сервер на **любом порту, кроме 3000** (микрофон обычно требует HTTPS или localhost):
   ```bash
   npx serve . -p 3333
   ```
   или `-p 8080`, `-p 5000` и т.д.
2. Откройте в браузере `http://localhost:3333` (или выбранный порт).
3. Разрешите доступ к микрофону.
4. Нажмите «Начать игру».
5. Слушайте ноту с «пианино» и пойте её, когда шарик подъезжает к стене — тогда он проедет дальше.

## Деплой на GitHub Pages (отдельный репозиторий)

Проект — статический сайт без сборки, подходит для публикации на GitHub Pages в **отдельном** репозитории.

1. Создайте **новый пустой репозиторий** на GitHub (без README, без .gitignore).
2. В каталоге проекта выполните:
   ```bash
   git init
   git add .
   git commit -m "Initial commit: Vocal Ball"
   git branch -M main
   git remote add origin https://github.com/<ВАШ_USERNAME>/<ИМЯ_РЕПО>.git
   git push -u origin main
   ```
3. В репозитории: **Settings → Pages**.
4. В разделе **Source** выберите **Deploy from a branch**.
5. В **Branch** выберите `main` и папку `/ (root)`, нажмите Save.
6. Через 1–2 минуты сайт будет доступен по адресу:
   `https://<ВАШ_USERNAME>.github.io/<ИМЯ_РЕПО>/`

Для работы микрофона в Production нужен **HTTPS** — на GitHub Pages он включён по умолчанию.

## MVP

- Один уровень с семью препятствиями по нотам До–Си.
- Неоновый дизайн, фоновая нота для текущего препятствия.
- Определение высоты голоса по микрофону (Web Audio API).

## Технологии

- HTML5 Canvas, CSS (неон), ванильный JS.
- Web Audio API: воспроизведение нот, анализ частоты с микрофона.
