# Vocal Ball — вокальная распевка

2D-платформер: шарик движется сам, препятствия проходимы только при пении нужной ноты (До–Ре–Ми–Фа–Соль–Ля–Си). Управление голосом через микрофон.

## Важно: порт 3000

**Порт 3000 занят другим приложением.** Локальный сервер для этого проекта **нельзя** запускать на порту 3000 — используйте любой другой порт (например 3333 или 8080).

## Звуки нот (MP3)

Ноты пианино проигрываются из папки `sounds/notes/`. Скопируйте туда свои MP3 (например из «Ноты пианино» в Загрузках). Имена файлов по ноте — латиница или кириллица:

- `C.mp3` или `До.mp3`, `D.mp3` или `Ре.mp3`, `E.mp3` или `Ми.mp3`, `F.mp3` или `Фа.mp3`, `G.mp3` или `Соль.mp3`, `A.mp3` или `Ля.mp3`, `B.mp3` или `Си.mp3`

Каждый файл — один такт ноты; он циклически повторяется, пока шарик не перейдёт к следующей ноте. Если файлов нет, используется синтезированный тон.

## Запуск локально

1. Поднимите сервер на **любом порту, кроме 3000** (микрофон обычно требует HTTPS или localhost):
   ```bash
   npx serve . -p 3333
   ```
   или `-p 8080`, `-p 5000` и т.д.
2. Откройте в браузере `http://localhost:3333` (или выбранный порт).
3. Разрешите доступ к микрофону.
4. Нажмите «Начать игру».
5. Слушайте ноту с «пианино» и пойте её, когда шарик подъезжает к стене — тогда он проедет дальше.

## Деплой на GitHub Pages

Репозиторий: [github.com/tunsuyokii/vocal-game](https://github.com/tunsuyokii/vocal-game).  
Сайт: **https://tunsuyokii.github.io/vocal-game/**

### Первая публикация

1. Создайте на GitHub пустой репозиторий **vocal-game**: [Create a new repository](https://github.com/new?name=vocal-game) (Public, без README/.gitignore).
2. В каталоге проекта выполните:
   ```powershell
   cd c:\Projects\vocal-game
   git remote add origin https://github.com/tunsuyokii/vocal-game.git
   git push -u origin main
   ```
3. В репозитории: **Settings → Pages** → Source: **GitHub Actions** (workflow `Deploy to GitHub Pages`).
4. После следующего push (или повторного запуска workflow) сайт будет доступен по ссылке выше.

Дальнейший деплой: при каждом `git push origin main` workflow автоматически публикует сайт.

Для работы микрофона в Production нужен **HTTPS** — на GitHub Pages он включён по умолчанию.

## MVP

- Один уровень с семью препятствиями по нотам До–Си.
- Неоновый дизайн, фоновая нота для текущего препятствия.
- Определение высоты голоса по микрофону (Web Audio API).

## Технологии

- HTML5 Canvas, CSS (неон), ванильный JS.
- Web Audio API: воспроизведение нот, анализ частоты с микрофона.
